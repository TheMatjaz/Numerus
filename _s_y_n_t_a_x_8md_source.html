3 Numerus nomenclature
4 ===============================================================================
5 
6 - **Roman char** or
7  **Roman character**:
8  a character from the latin alphabet that represents a numeric value, for
9  instance `C` means `100`.
10 - **Numeral** or
11  **Roman numeral**:
12  a sequence of roman characters that have to be summed to represent a more
13  complex value.
14 - **Value**:
15  usually this word indicates the value a roman numeral represents
16 - **Integer part**:
17  the integer part of a value is the part before the decimal separator
18  (usually the dot or comma) or a value without considering the twelfths.
19 - **Long numeral**:
20  a Numerus syntax to indicate numerals that hold integer values outside
21  `[-3999, 3999]` that are written with underscores representing the
22  overlined notation - where overlined characters have their values
23  multiplied by 1000.
24 - **Float numeral**:
25  a Numerus syntax to indicate numerals that hold a non-zero number of
26  twelfths as well, making them approximate a floating point value.
27 - **CLI**:
28  stands for _Command Line Interface_, a Numerus shell where any roman
29  numeral can be typed to be converted to a value or the other way around.

32 Roman numerals syntax (briefly)
33 ===============================================================================
34 
35 This is a short explanation on how roman numerals work. For a different
36 one, check the
37 [Wikipedia article](https://en.wikipedia.org/wiki/Roman_numerals) on them.
38 
39 
40 Character values
41 ----------------------------------------
42 
43 Roman numerals are sequences of characters from the latin alphabet and each
44 represents a different value:
45 
46 | Char | Value |
47 |------|-------|
48 | M | 1000 |
49 | D | 500 |
50 | C | 100 |
51 | L | 50 |
52 | X | 10 |
53 | V | 5 |
54 | I | 1 |
55 | S | 1/2 |
56 | . | 1/12 |
57 
58 When a character with a smaller value is placed before a character with
59 bigger value (except for `S` and `.`), they have to be interpreted together
60 as one and their value have to be computed by subtraction _bigger - smaller_,
61 resulting in:
62 
63 | Char | Value |
64 |------|-------|
65 | CM | 900 |
66 | CD | 400 |
67 | XC | 90 |
68 | XL | 40 |
69 | IX | 9 |
70 | IV | 4 |
71 
72 The rules to read and write roman numerals are:
73 
74 1. When a character with smaller value stand before a character with bigger
75  value, subtract the smaller from the bigger, as shown above.
76 2. For everything else, sum from the left to the right.
77 
78 An example:
79 
80 ```
81 CDLXXXVIIS... = CD + L + XXX + V + II + S + ...
82  / \ | / | \ | |\ | /|\
83  = 500 - 100 + 50 + 10+10+10 + 5 + 1+1 + 1/2 + 3/12
84 
85  = 487 + 3/4 = 487.75
86 ```

89 Standard roman numeral syntax
90 ----------------------------------------
91 
92 The syntax of a standard roman numeral requires a specific sequence of
93 characters as in the following points:
94 
95 - 0-3 M
96 - 0-1 CM or 0-1 CD or ( 0-1 D and 0-3 C )
97 - 0-1 XC or 0-1 XL or ( 0-1 L and 0-3 X )
98 - 0-1 IX or 0-1 IV or ( 0-1 V and 0-3 I )
99 
100 Please notice that there are no `S` or `.` in it. This allows writing integer
101 values in `[1, 3999]`. In a regex form it would be:
102 
103 ```
104 M{0,3}(CM|CD|D?C{0,3})(XC|XL|L?X{0,3})(IX|IV|V?I{0,3})
105 ```

109 Numerus syntax extensions
110 ===============================================================================
111 
112 Numerus add more syntax rules, while keeping compatibility with the standard
113 roman numeral syntax, with the goal of writing numerals with more values.
114 
115 
116 Zero
117 ----------------------------------------
118 
119 Numerus uses the string `NULLA` (as stored in the global constant variable
120 `NUMERUS_ZERO`) for the numeral of value 0 (zero). The word _nulla_ means
121 _nothing_ in latin.
122 
123 This allows writing integer values in `[0, 3999]`.
124 
125 
126 Negative values
127 ----------------------------------------
128 
129 Numerus uses the minus `-` sign at the beginning of a roman numeral to indicate
130 that its value is negative. This allows writing integer values in
131 `[-3999, 3999]`.
132 
133 An example:
134 
135 ```
136  II = 2
137 -II = -2
138 -MMCCCXLV = -2345
139 -NULLA = 0
140 
141 ```

144 Long numerals
145 ----------------------------------------
146 
147 In ancient Rome, positive values bigger than 3999 were indicated with an
148 overlining, called _vinculum_. The overlined characters have their value
149 multiplied by 1000.
150 
151 An example:
152 
153 ```
154 ___
155 MIIXVI = 1002016
156 ```
157 
158 Numerus adopted this syntax with a difference: instead of actually overlining
159 the characters, making them impossible to parse, encloses the characters that
160 _should_ be overlined with _underscores_ `_`. This allows a simple record of
161 characters with values that have to be multiplied by 1000. Creating an actually
162 overlined numeral (a two-line string with underscores in the first line) is
163 possible with the numeral prettifying function.
164 
165 These are called _long numerals_ in the Numerus nomenclature, resembling the
166 name of a `long int` value they represent. This allows writing integer values in
167 `[-3999999, 3999999]`.
168 
169 Long numerals have have the following syntax:
170 
171 - the overlined part (between underscores) has standard roman numeral syntax
172 - the non-overlined part (after the underscores) has standard roman numeral
173  syntax without `M`
174 
175 An example:
176 
177 ```
178  _MII_XVI = 1002016
179 -_IV_I = -4001
180  _MCMLII_ = 1952000
181 ```

183 Float numerals
184 ----------------------------------------
185 
186 Roman numerals provide fractional values in _twelfths (1/12)_ using the
187 character central dot `â€¢` to indicate a _twelfth_. Numerus uses the normal dot
188 (period) `.` instead for simplicity. The character `S` is used to indicate one
189 _half (6/12)_ - `S` stands for _semis_, which means _half_ in latin.
190 
191 Numerus implements that feature allowing a fractional part in all roman numerals
192 (except `NULLA`) just by adding `S` or `.` at the end of the numeral. These are
193 called _float numerals_ in the Numerus nomenclature, resembling the name of a
194 `float` value they approximate. This allows writing all twelfths values in
195 `[-3999999.91666, 3999999.91666]`.
196 
197 Float numerals have have the following syntax:
198 
199 - at the end of any roman numeral append `S` to indicate a fractional part of
200  one half (6/12) or more, if that is the case;
201 - append up to five `.` to indicate additional 1 to 5 twelfths.
202 - instead of indicating 12/12, switch to the next roman numeral instead of
203  appending another twelfth.
204 
205 Please note that a float numeral can be at the same time a long numeral and
206 vice-versa. The nomenclature is a pure formality to indicate the capability of
207 the numeral.

210 Regex matching them all
211 ---------------------------------------
212 
213 Just for the documentation a regex matching all long, float, negative and
214 standard roman numerals but not (`-`)`NULLA` is:
215 
216 ```
217 -?((_M{0,3}(CM|CD|D?C{0,3})(XC|XL|L?X{0,3})(IX|IV|V?I{0,3})_)|M{0,3})(CM|CD|D?C{0,3})(XC|XL|L?X{0,3})(IX|IV|V?I{0,3})S?\\.{0,5}
218 ```
