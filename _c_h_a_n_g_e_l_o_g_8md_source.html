3 v2.0.0
4 ===============================================================================
5 
6 
7 Added
8 ----------------------------------------
9 
10 1. Support for roman numerals with integer values outside of `[-3999, 3999]`
11  emulating the overlined notation of characters with values multiplied
12  by 1000. Numerus uses underscores to indicate overlined parts of a numeral,
13  which in this case is called _long roman numeral_.
14 2. Support for roman numerals with twelfths as an approximation of fractional
15  parts of the value using characters `S` for `1/2` and dot `.` for `1/12`.
16  The numerals in this case are called _float roman numerals_.
17 3. More constants indicating the extremes of the value range for different
18  types of roman numerals.
19 4. Numeral analysis functions:
20  1. check if a numeral has value zero;
21  2. check if a numeral is a _long numeral_;
22  3. check if a numeral is a _float numeral_;
23  4. count the number of roman character - and find any illegal ones.
24 5. Twelfth management functions:
25  1. convert a double from and to an integer value plus a number of twelfths;
26  2. shorten an integer value and a number of twelfths.
27 6. Human readability functions:
28  1. prettify a long roman numerals with actual overlining;
29  2. prettify an integer value and a number of twelfths with shortening;
30  3. error code explanation generation.
31 7. _Command Line Interface_ (CLI) to quickly convert any value; also parses
32  any command line arguments (main arguments) as a CLI command, allowing easy
33  pipelining of the executable in the system shell.
34 
35 
36 Changed
37 ----------------------------------------
38 
39 1. Roman numerals' syntax check is performed without regex-es, making it ~75%
40  faster, while making the conversion function support the new roman numeral
41  formats as well.
42 2. Because of the previous point, it's possible to catch a more precise error
43  when the syntax is wrong.
44 3. Error codes are used much much more, they have different (clearer) names
45  and better values overall. They are returned as a passed parameter pretty
46  much everywhere, making the user able to understand the exact problem.
47 4. Function names have changed to a more understandable version.
48 5. All code is completely rewritten, carefully optimized and (trying to be)
49  foolproof.
50 
51 
52 Fixed
53 ----------------------------------------
54 
55 1. All roman numerals are now case insensitive.
56 2. Roman numerals are initially checked if NULL or empty. Any leading
57  whitespace is ignored.
58 3. `malloc()` failures are controlled and returned in the error code.
