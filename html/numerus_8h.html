#define NUMERUS_ERROR_CANNOT_COMPARE   104;

numerus_compare_value() comparision failed.

It could not compare the two roman numerals because at least one of them cannot be converted to short so it has no value.

#define NUMERUS_ERROR_OUT_OF_RANGE   101;

Integer out of conversion range.

numerus_short_to_roman() cannot convert the passed value because it exceed the possible values a roman numeral may represent, which are the integers in the interval [NUMERUS_MIN_VALUE, NUMERUS_MAX_VALUE].

#define NUMERUS_ERROR_REGEXEC   102;

regexec() internal error.

Happens while checking the syntax correctness of a roman numeral to be converted to short by numerus_roman_to_short(). Check regerror() and the stderr for more information.

char** numerus_allocate_all_romans ( short int  include_negatives)

Allocates all roman numerals and their values in memory for fast conversions from value to roman numeral.

It creates an array of pointers to strings, char*. Each char* points to the roman numeral with the same value as the index of the char* in the array. The returned pointer to the array points to the value 0.

Example structure:

*  i  | p --> numeral
* ----|---------------------
* -1  | * --> "-I"
*  0  | * --> "NULLA"
*  1  | * --> "I"
*

Example usage:

char **all_romans = numerus_allocate_all_romans(1);
char *fortytwo = all_romans[42];
Parameters
include_negativesset it to 0 to create the array from 0 to NUMERUS_MAX_VALUE or to anything else to create it from NUMERUS_MIN_VALUE to NUMERUS_MAX_VALUE
Returns
the address of the value 0 (which points to NUMERUS_ZERO) in the array or NULL if malloc() fails to allocate the array

int numerus_is_roman ( char *  roman)

Verifies if the passed string is a correct roman numeral.

Performs syntax check of the passed roman numeral by checking it against a regex compiled from NUMERUS_SYNTAX_REGEX_STRING. It is case insensitive. The compilation is once for all subsequent calls of the function during runtime. The regex compilation status is dropped since NUMERUS_SYNTAX_REGEX_STRING is a correct hard coded constant.

Parameters
*romanstring containing a roman numeral to check
Returns
int 1 if has correct roman syntax, 0 if it does not and in case of regex errors.

Flags in regcomp():

  • REG_NOSUB: does not save subexpressions (groups), only reports the success or failure of compiling the regex
  • REG_ICASE: ignores the case, making the regex case insensitive
  • REG_EXTENDED: uses the extended POSIX standard regular expressions, which are required for the regex structure

short numerus_roman_to_short ( char *  roman)

Converts a roman numeral to a short int.

It is case insensitive and accepts negative roman numerals. If the numeral cannot be converted, it means it has wrong syntax. In that case a value bigger than NUMERUS_MAX_VALUE is returned and the error code NUMERUS_ERROR_WRONG_SYNTAX is stored in numerus_error_code.

Parameters
*romanstring with a roman numeral
Returns
short value of the roman numeral or a value bigger than NUMERUS_MAX_VALUE in case of error
char* numerus_short_to_roman ( short int  arabic)

Converts a short int to a roman numeral.

It allocates a string with the roman numerals long just as required and returns a pointer to it. If the short is outside of [NUMERUS_MIN_VALUE, NUMERUS_MAX_VALUE], the conversion is impossible.

Returns
pointer to a string containing the roman numeral, NULL if the short is out of range.

short int numerus_error_code

Global error code variable to store any errors during conversions.

It may contain any of the ROMAN_ERROR_* error codes or NUMERUS_OK.

const short int NUMERUS_MAX_LENGTH

Maximum length of a roman numeral string including the null terminator.

The roman numeral "-MMMDCCCLXXXVIII" (value: -3888) + \0 is a string long 16+1 = 17 chars.
